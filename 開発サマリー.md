# 開発サマリー

このファイルは、「日米Amazonアービトラージツール」のこれまでの開発内容と今後のタスクをまとめたものです。後から作業を再開する際のガイドとしてお使いください。

## 1. 概要

- **目的:** 日本のAmazonの商品を米国のAmazonで無在庫販売するためのWebアプリケーション。
- **技術構成:**
  - **フロントエンド:** React, TypeScript, Material-UI
  - **バックエンド:** Node.js, Express
- **プロジェクトルート:** `amazon-arbitrage-tool/`

## 2. 起動方法

**注意:** フロントエンドとバックエンドは、それぞれ別のターミナルで起動する必要があります。

### バックエンド (APIサーバー)

1. ターミナルで `amazon-arbitrage-tool/backend` ディレクトリに移動します。
2. `npm start` コマンドを実行します。
3. `Server is running on http://localhost:3001` と表示され、プロセスが待機状態になれば成功です。

### フロントエンド (画面)

1. ターミナルで `amazon-arbitrage-tool/frontend` ディレクトリに移動します。
2. `npm start` コマンドを実行します。
3. 自動的にブラウザで `http://localhost:3000` が開きます。

## 3. 設定ファイル

### APIキーと認証情報

- **場所:** `amazon-arbitrage-tool/backend/.env`
- **内容:** Amazon SP-APIの各種認証キーを設定します。

### アプリケーション設定

- **場所:** `amazon-arbitrage-tool/backend/config/settings.json`
- **内容:** 送料、手数料、為替レート、在庫監視の閾値、除外リスト、**リードタイムバッファ(n値)** などの設定が保存されます。

## 2026/05/28 開発サマリー（最新）

### 1. SP-API Pricing「Access Denied」エラーの解消
- **不具合:** 米国の商品リサーチ中に日本の価格を取得しようとすると「Access to requested resource is denied.」が発生していた。
- **原因:** 
    - 日本リージョンのエンドポイントURLが不完全だった。
    - セラーセントラルで「Pricing」の役割（Role）を追加した後に、アプリの再認可（トークンの更新）が行われていなかった。
- **修正内容:** 
    - `sp-api-client.js` を修正し、リージョンごとに正しいエンドポイント（`fe` / `na`）を明示的に指定。
    - アプリの再連携を促し、新しい権限を含むトークンを保存。

### 2. リサーチデータの精度向上（重量・カテゴリ）
- **カテゴリ取得の修正:** `item.productType` ではなく `item.productTypes`（配列）を参照するように修正し、`TOY_FIGURE` などの正確なカテゴリ名が表示可能に。
- **重量表示の改善:** CSVやUIで `{"unit":"kilograms","value":0.259}` のように生JSONが表示されていた問題を解決。「0.259 kilograms」等の読みやすい形式へ変換。
- **API取得漏れ対策:** AmazonカタログAPIが1回のリクエストで10件しか返さない仕様に合わせ、チャンクサイズを10に変更し `pageSize: 10` を明示。これによりデータの「歯抜け」を解消。

### 3. 出品ロジックの抜本的強化（出品成功率の向上）
- **米国在庫なし商品への対応:** 米国価格が 0（ライバル不在）の商品を「スキップ」せず、日本価格から逆算（1.5倍 + 3000円）して自動でドル価格を計算し出品するロジックを実装。
- **出品情報なし（！）エラーの解消:**
    - カタログAPIから取得した正確な `productType`（例: `NON_RIDING_TOY_VEHICLE`）を送信するように強化。
    - 在庫数が 0 にならないよう、最低でも `1` を確実にセットするように修正。
- **既存出品の上書き許可:** 既に出品済みのASINもスキップせず、最新の計算価格と在庫数で上書き（Update）するように変更。これによりセラーセントラル上の不完全な出品情報の修復が可能に。

### 4. データベース保存エラーの修正
- **JSON形式エラー:** PostgreSQL保存時に `invalid input syntax for type json` が発生していた問題を、`JSON.stringify()` の適用により解決。

## 次世代トップセラー戦略 (2026年版提案)

最新のリサーチに基づき、アカウント停止リスクを最小化しつつ収益を最大化するための戦略を提案します。

### 1. Amazon AI監視「Risk-Shield」対策
- **超高速在庫同期:** 在庫切れによる注文キャンセルは即アカウント停止に直結するため、SP-APIのNotifications APIを活用したイベント駆動型の同期（15分以内）への移行。
- **価格変動の安全性:** 24時間以内に20%以上の価格変更を行う際は「人間による承認」を挟むUIの実装（2026年新ポリシー準拠）。

### 2. 規約遵守 (Compliance) の徹底
- **Seller of Recordの維持:** 卸・メーカー仕入れへのシフトと、米国のPrepセンター（発送代行）を活用した「他社ロゴ排除」の徹底。
- **出品前AIスキャン:** 商品説明や画像に禁止キーワード（小売店名やロゴ）が含まれていないかを自動検知する機能の追加。

### 3. 収益最大化 (Profit Optimization)
- **カート価格 (Buy Box) 自動追従:** 単なるコスト積み上げ価格ではなく、米国の競合価格をリアルタイム取得し、利益限界まで自動で合わせるロジックの実装。
- **Keepa売れ筋分析:** 過去30日の販売実績（Drops数）を取得し、回転率の高い商品に絞って出品・在庫補充を行う「高回転・低在庫」モデルへの転換。
- **カテゴリ別関税の自動適用:** 商品カテゴリに応じた正確な関税率計算により、想定外のコスト発生（赤字）を防止。
