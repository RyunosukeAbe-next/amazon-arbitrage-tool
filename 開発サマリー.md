# 開発サマリー

このファイルは、「日米Amazonアービトラージツール」のこれまでの開発内容と今後のタスクをまとめたものです。後から作業を再開する際のガイドとしてお使いください。

## 1. 概要

- **目的:** 日本のAmazonの商品を米国のAmazonで無在庫販売するためのWebアプリケーション。
- **技術構成:**
  - **フロントエンド:** React, TypeScript, Material-UI
  - **バックエンド:** Node.js, Express
- **プロジェクトルート:** `amazon-arbitrage-tool/`

## 2. 起動方法

**注意:** フロントエンドとバックエンドは、それぞれ別のターミナルで起動する必要があります。

### バックエンド (APIサーバー)

1. ターミナルで `amazon-arbitrage-tool/backend` ディレクトリに移動します。
2. `npm start` コマンドを実行します。
3. `Server is running on http://localhost:3001` と表示され、プロセスが待機状態になれば成功です。

### フロントエンド (画面)

1. ターミナルで `amazon-arbitrage-tool/frontend` ディレクトリに移動します。
2. `npm start` コマンドを実行します。
3. 自動的にブラウザで `http://localhost:3000` が開きます。

## 3. 設定ファイル

### APIキーと認証情報

- **場所:** `amazon-arbitrage-tool/backend/.env`
- **内容:** Amazon SP-APIの各種認証キーを設定します。

### アプリケーション設定

- **場所:** `amazon-arbitrage-tool/backend/config/settings.json`
- **内容:** 送料、手数料、為替レート、在庫監視の閾値、除外リスト、**リードタイムバッファ(n値)** などの設定が保存されます。

## 2026/04/28 本日の修正・進捗まとめ

### 1. 出品エラーの解消とカテゴリー指定の最適化
- **ProductTypeの修正:** `getCatalogItemsByAsins` でカテゴリー名がオブジェクトとして取得されていた問題を修正。
- **出品の安定化:** 特定のカテゴリー（例: `TOWEL`）を指定するとAmazonから詳細な商品情報を求められるため、汎用的な `PRODUCT` に固定して送信するように変更。これにより、在庫更新がエラーで止まるリスクを回避。
- **オファー更新の明示:** 既存ASINへの相乗り出品であることを示す `LISTING_OFFER_ONLY` 属性を正しく設定。

### 2. 在庫数（数量）の動的反映
- **日本Amazonの在庫追従:** 日本の出品者数（`jpSellerCount`）を取得し、これを出品時の在庫数（`quantity`）として設定するロジックを実装。
- **安全装置:** 日本の在庫が0や不明な場合でも、Amazon側で出品が無効化されないよう、出品時には最低でも「1」の在庫を設定して送信。

### 3. リードタイム（お届け日）の自動計算
- **設定追加:** フロントエンド「設定」タブに「リードタイムバッファ (n値)」を追加。
- **自動計算:** `日本の出荷作業日数 + n値` を計算し、米国のリードタイムとして設定。デフォルトは日本2日 + バッファ3日 = 計5日。

### 4. 自動管理機能（価格・在庫）の強化
- **価格調整機能 (`price-adjuster.js`):** 15分おきの価格更新時に、価格だけでなく日本の最新在庫数も再取得してAmazonに反映させるように強化。
- **在庫監視機能 (`inventory-watcher.js`):** 1時間おきに在庫切れ・赤字転落をチェックし、基準外の商品は自動で出品を取り下げる（削除する）ことを確認。
